# language: zh-TW

@US-CMPM-REQ-002 @request-management @P0
功能: 需求瀏覽
  身為平台使用者
  我想要能夠瀏覽、搜尋和篩選平台上發布的各類需求
  以便找到與我相關的合作機會或能提供幫助的對象

  背景:
    假設我已成功登入社團企業媒合平台
    而且系統中已存在多筆不同類型的需求

  @list-view
  場景: 需求列表瀏覽
    當我點擊頁面上的"瀏覽需求"按鈕
    那麼系統應該顯示需求列表頁面
    而且我應該能看到分頁控制器
    而且每頁應該顯示20項需求

    當列表中有需求時
    那麼對於每個需求項目，我應該能看到：
      | 需求標題 |
      | 發布者類型 |
      | 發布日期 |
      | 需求類型 |
      | 截止日期 |

    當沒有任何需求時
    那麼系統應該顯示"目前沒有符合條件的需求"訊息

  @view-mode-switch
  場景: 切換瀏覽模式
    當我進入需求列表頁面
    那麼系統應該默認顯示"列表"模式
    
    當我點擊"卡片模式"按鈕
    那麼系統應該切換到卡片視圖
    而且需求應該以卡片格式呈現
    
    當我點擊"列表模式"按鈕
    那麼系統應該切換回列表視圖
    而且需求應該以表格列表格式呈現

  @request-detail
  場景: 查看需求詳情
    當我在需求列表中點擊某個需求項目"專題合作夥伴招募"
    那麼系統應該跳轉到該需求的詳情頁面
    而且我應該能看到完整的需求資訊，包括：
      | 完整描述 |
      | 標籤 |
      | 附件 |
      | 發布者資訊 |
    而且我應該看到"申請合作"按鈕
    
    當需求已過期時
    那麼系統應該顯示"此需求已過期"訊息
    而且"申請合作"按鈕應該被禁用
    
    當需求已達合作上限時
    那麼系統應該顯示"此需求已達合作上限"訊息
    而且"申請合作"按鈕應該被禁用

  @request-favorite
  場景: 收藏需求
    當我在需求詳情頁面點擊"加入收藏"按鈕
    那麼系統應該顯示"已加入收藏"確認訊息
    而且該按鈕應變更為"已收藏"狀態
    
    當我再次點擊"已收藏"按鈕
    那麼系統應該顯示"已從收藏中移除"確認訊息
    而且該按鈕應變回"加入收藏"狀態
    
    當我進入"我的收藏"頁面
    那麼我應該看到我收藏的所有需求列表

  @search-function
  場景大綱: 需求搜尋功能
    當我在需求列表頁的搜尋框中輸入"<關鍵詞>"
    而且我點擊搜尋按鈕
    那麼系統應該<結果>
    
    例子:
      | 關鍵詞       | 結果                                                   |
      | 設計         | 顯示標題或描述中包含"設計"的需求，並高亮關鍵詞        |
      | 程式 開發    | 顯示標題或描述中同時包含"程式"和"開發"的需求          |
      | 無效關鍵詞xyz | 顯示"找不到符合『無效關鍵詞xyz』的需求，請嘗試其他關鍵詞" |
      | a            | 顯示"請輸入至少2個字元"的提示                          |
      |              | 顯示"請輸入搜尋關鍵詞"的提示                           |

  @filter-and-sort
  場景: 篩選與排序功能
    當我進入需求列表頁面
    那麼我應該看到篩選選項區域
    
    當我選擇需求類型篩選為"資源需求"
    那麼系統應該只顯示類型為"資源需求"的需求
    
    當我選擇發布角色篩選為"企業"
    那麼系統應該只顯示由"企業"角色發布的需求
    
    當我同時選擇需求類型"資源需求"和標籤"設計"
    那麼系統應該只顯示同時滿足這兩個條件的需求
    
    當應用篩選後沒有符合條件的需求
    那麼系統應該顯示"沒有符合篩選條件的需求"
    
    當我點擊"最新發布"排序選項
    那麼系統應該按發布時間由新到舊排序需求
    
    當我點擊"即將截止"排序選項
    那麼系統應該按截止日期由近到遠排序需求
    
    當我點擊"儲存此篩選"按鈕
    那麼系統應該提示我輸入篩選名稱
    當我輸入名稱"我的企業資源篩選"並確認
    那麼系統應該顯示"已儲存篩選條件"確認訊息

  @recommendation
  場景: 個人化推薦功能
    當我首次登入平台並進入需求瀏覽頁面
    那麼我應該在頁面頂部看到"為您推薦"區塊
    而且系統應該顯示"隨著您的使用，推薦將更加精準"提示
    
    當我有足夠的平台使用記錄後
    那麼"為您推薦"區塊應該顯示基於我的角色和互動記錄的相關需求
    
    當我查看某個需求詳情頁面
    那麼我應該在頁面底部看到"更多類似需求"區塊
    而且該區塊應該顯示與當前需求相似的其他需求
