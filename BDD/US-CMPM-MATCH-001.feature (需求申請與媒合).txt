@US-CMPM-MATCH-001 @match-request @P0
功能: 需求申請與媒合
  身為平台使用者
  我想要能夠申請感興趣的需求並追蹤申請進度
  以便與發布需求的一方建立合作關係，達成雙方的目標

  背景:
    假設我已成功登入社團企業媒合平台
    而且系統中已存在多筆不同類型的需求

  @request-application
  場景大綱: 提交需求申請
    假設我正在瀏覽需求"<需求標題>"的詳情頁面
    而且該需求目前狀態為"開放申請中"
    
    當我點擊"申請合作"按鈕
    那麼系統應該顯示申請表單
    
    當我填寫意向說明"<意向說明>"
    而且我從預設模板選擇"<申請模板>"
    而且我上傳相關證明文件"<文件名稱>"
    當我點擊"提交申請"按鮮
    那麼系統應該<結果>

    例子:
      | 需求標題                   | 意向說明                                           | 申請模板           | 文件名稱           | 結果                                          |
      | 校園宣傳活動設計合作       | 我是設計系的學生，有豐富的活動設計經驗              | 學生專業合作模板   | portfolio.pdf      | 顯示"申請已提交，等待對方回應"訊息           |
      | 企業實習機會               | 我熟悉程式開發，希望獲得實習機會                  | 實習申請模板       | resume.pdf         | 顯示"申請已提交，等待對方回應"訊息           |
      | 活動場地贊助               |                                                    | 資源申請模板       | proposal.pdf       | 顯示"請說明您為何適合此需求"錯誤訊息          |
      | 校園宣傳活動設計合作       | 我曾設計過多個活動宣傳物，希望與貴方合作          | 學生專業合作模板   | portfolio.pdf      | 顯示"您已申請此需求"提示訊息                 |

  @application-status
  場景: 申請狀態追蹤
    假設我已提交過多個需求申請
    
    當我點擊導航菜單中的"我的申請"選項
    那麼系統應該顯示我的申請列表頁面
    而且我應該看到我所有已提交的申請記錄
    而且每條記錄應該顯示以下信息：
      | 需求標題 |
      | 申請日期 |
      | 申請狀態 |
      | 最後更新時間 |
    
    當我的某個申請狀態為"待審核"
    那麼該申請應該顯示"等待審核中"標籤
    
    當我的某個申請狀態更新為"已查看"
    那麼系統應該發送通知"您的申請已被查看"
    而且該申請應該顯示"對方已查看"標籤
    
    當我的某個申請狀態更新為"已接受"
    那麼系統應該發送通知"您的申請已被接受"
    而且該申請應該顯示"申請成功"標籤
    
    當我的某個申請狀態更新為"已拒絕"
    那麼系統應該發送通知"您的申請未被採納"
    而且該申請應該顯示"未獲採納"標籤

  @view-application-detail
  場景: 查看申請詳情
    假設我在"我的申請"列表頁面
    
    當我點擊某個申請記錄"校園活動合作"
    那麼系統應該顯示該申請的詳情頁面
    而且我應該能看到：
      | 申請表單內容 |
      | 上傳的文件 |
      | 申請時間 |
      | 申請狀態及歷史 |
      | 對方回覆(如有) |
    
    當申請狀態為"待審核"
    那麼我應該看到"撤回申請"按鈕
    
    當我點擊"撤回申請"按鈕
    那麼系統應該顯示確認對話框
    當我確認撤回申請
    那麼系統應該顯示"申請已撤回"訊息
    而且該申請狀態應變更為"已撤回"

  @application-response
  場景: 需求發布者處理申請
    假設我是需求"校園講座支援"的發布者
    而且該需求已收到多個申請
    
    當我點擊"我的需求"中的"校園講座支援"
    那麼我應該看到該需求的申請列表
    而且我應該看到每個申請的摘要信息
    
    當我點擊查看某個申請詳情
    那麼系統應該顯示完整的申請信息
    
    當我點擊"接受申請"按鈕
    那麼系統應該顯示確認對話框
    當我確認接受並填寫附加說明"期待與您合作"
    那麼系統應該顯示"已接受申請，系統已通知對方"訊息
    而且該申請狀態應變更為"已接受"
    
    當我點擊"拒絕申請"按鈕
    那麼系統應該顯示確認對話框
    當我確認拒絕並選擇拒絕原因"已找到其他合作方"
    那麼系統應該顯示"已拒絕申請，系統已通知對方"訊息
    而且該申請狀態應變更為"已拒絕"
    
    當我點擊"批量操作"按鈕
    那麼系統應該顯示批量操作選項
    當我選中多個申請並選擇"批量拒絕"
    那麼系統應該顯示確認對話框
    當我確認批量拒絕並選擇拒絕原因"需求已滿足"
    那麼系統應該顯示"已批量拒絕申請，系統已通知對方"訊息

  @match-confirmation
  場景: 初步媒合確認
    假設我的申請"設計合作提案"已被接受
    而且我收到了"您的申請已被接受"通知
    
    當我點擊該通知或進入申請詳情頁面
    那麼系統應該顯示初步媒合確認頁面
    而且我應該看到對方提供的額外說明或要求
    
    當我點擊"確認合作"按鈕
    那麼系統應該顯示合作細節確認表單
    當我填寫我方聯絡人"王小明"
    而且我填寫聯絡方式"0912345678"
    而且我確認合作內容和時間
    當我點擊"確認提交"按鈕
    那麼系統應該顯示"您已確認合作細節，等待對方確認"訊息
    
    當對方也確認合作細節後
    那麼系統應該顯示"合作已確認，進入執行階段"訊息
    而且系統應該創建新的合作關係並導向合作管理頁面

  @recommendation-matching
  場景: 自動推薦功能
    當我登入平台並訪問首頁
    那麼我應該在"為您推薦"區域看到系統根據我的資料和歷史匹配的需求
    
    當我發布新需求"尋找設計合作夥伴"
    那麼系統應該分析我的需求並在"推薦申請者"區域顯示匹配度高的潛在合作者
    而且每個推薦應該顯示匹配理由
    
    當我點擊某個推薦的合作者
    那麼系統應該顯示該合作者的詳細資料和匹配度分析
    
    當我點擊"發送合作邀請"按鈕
    那麼系統應該顯示邀請表單
    當我填寫邀請訊息並確認發送
    那麼系統應該顯示"合作邀請已發送"訊息
