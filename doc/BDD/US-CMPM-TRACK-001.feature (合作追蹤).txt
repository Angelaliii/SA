# language: zh-TW

@US-CMPM-TRACK-001 @collaboration-tracking @P1
功能: 合作追蹤
  身為已建立合作關係的平台使用者
  我想要能夠追蹤合作進度、設定里程碑並確認資源交付
  以便清楚掌握合作狀態，確保專案按計劃進行並達成預期目標

  背景:
    假設我已成功登入社團企業媒合平台
    而且我已與"創新科技有限公司"建立合作關係

  @collaboration-list
  場景: 合作項目管理
    當我點擊導航菜單中的"我的合作"選項
    那麼系統應該顯示我的合作列表頁面
    
    當我有進行中的合作項目時
    那麼我應該看到分類選項卡，包含"進行中"、"已完成"和"已取消"
    而且在"進行中"分類下，我應該看到所有正在進行的合作項目
    而且每個合作項目應該顯示：
      | 合作對象 |
      | 需求名稱 |
      | 開始日期 |
      | 預計完成日期 |
      | 當前狀態 |
    
    當我沒有任何合作項目時
    那麼系統應該顯示"您目前沒有進行中的合作項目"訊息
    而且提供"瀏覽需求"按鈕引導我尋找合作機會
    
    當我有即將到期的合作項目
    那麼系統應該在合作列表頂部顯示提醒"您有X個合作項目即將到期"
    而且這些項目應該以醒目方式標示

  @collaboration-detail
  場景: 查看合作詳情
    當我在合作列表中點擊特定合作項目"校園講座合作"
    那麼系統應該顯示該合作的詳情頁面
    而且我應該看到以下信息：
      | 合作基本信息（雙方信息、開始日期、預計結束日期等） |
      | 合作內容摘要 |
      | 進度狀態 |
      | 里程碑列表 |
      | 資源交付記錄 |
      | 問題列表 |
    
    當我點擊"編輯合作信息"按鈕
    那麼系統應該允許我更新可編輯的合作信息
    當我提交更新
    那麼系統應該保存變更並通知合作方

  @milestone-management
  場景: 里程碑設定與追蹤
    假設我正在查看合作"校園講座合作"的詳情頁面
    
    當我點擊"添加里程碑"按鈕
    那麼系統應該顯示添加里程碑表單
    
    當我填寫以下里程碑信息：
      | 里程碑名稱 | 講座內容定稿 |
      | 預計完成日期 | 2025-05-15 |
      | 負責方 | 雙方 |
      | 描述 | 確定講座主題、大綱和所有演講內容 |
    而且我點擊"保存"按鈕
    那麼系統應該顯示"新里程碑已新增"確認訊息
    而且該里程碑應該添加到里程碑列表中
    
    當里程碑"講座內容定稿"完成時
    而且我點擊該里程碑旁的"標記完成"按鈕
    那麼系統應該請求確認
    當我確認完成並提供完成說明"已完成所有講座內容設計"
    那麼系統應該通知合作方確認
    
    當合作方確認里程碑完成
    那麼系統應該更新里程碑狀態為"已完成"
    而且通知雙方"『講座內容定稿』已完成"
    
    當系統檢測到里程碑已超過預定日期但未完成
    那麼系統應該顯示警告"合作進度已落後計劃X天"
    而且向雙方發送提醒通知

  @resource-delivery
  場景大綱: 資源交付確認
    假設我正在查看合作"設計合作專案"的詳情頁面
    
    當我點擊"記錄資源交付"按鈕
    那麼系統應該顯示資源交付表單
    
    當我填寫以下資源交付信息：
      | 資源名稱 | <資源名稱> |
      | 資源類型 | <資源類型> |
      | 交付描述 | <交付描述> |
    而且我上傳交付證明"<證明文件>"
    而且我點擊"提交"按鈕
    那麼系統應該<結果>

    例子:
      | 資源名稱           | 資源類型 | 交付描述                           | 證明文件       | 結果                                              |
      | 設計素材包         | 檔案     | 包含所有設計素材和原始檔案         | design.zip     | 通知接收方"有新的資源交付等待確認"                |
      | 贊助款項           | 資金     | 已完成第一期款項轉帳               | receipt.pdf    | 通知接收方"有新的資源交付等待確認"                |
      | 技術諮詢           | 服務     | 已提供2小時的技術咨詢服務          | timesheet.pdf  | 通知接收方"有新的資源交付等待確認"                |
      |                    | 檔案     | 包含所有必要文件                   | files.zip      | 顯示"請輸入資源名稱"錯誤訊息                      |
      | 設計成果           | 檔案     |                                   | result.pdf     | 顯示"請提供交付描述"錯誤訊息                      |

  @delivery-confirmation
  場景: 接收方確認資源交付
    假設我是資源接收方
    而且我收到了"有新的資源交付等待確認"通知
    
    當我點擊該通知
    那麼系統應該顯示資源交付詳情
    
    當我檢查了交付內容並確認無誤
    而且我點擊"確認接收"按鈕
    那麼系統應該顯示"資源交付已確認"訊息
    而且通知資源提供方"資源交付已確認"
    
    當我檢查交付內容發現問題
    而且我點擊"提出異議"按鈕
    那麼系統應該顯示異議表單
    當我填寫異議原因"檔案無法打開，可能已損壞"
    而且我點擊"提交異議"按鈕
    那麼系統應該顯示"交付確認有異議，請查看詳情"
    而且通知資源提供方查看異議詳情

  @collaboration-status
  場景: 更新合作狀態
    假設我正在查看合作"網站開發專案"的詳情頁面
    
    當我點擊"更新合作狀態"按鈕
    那麼系統應該顯示狀態更新對話框
    而且我應該看到以下狀態選項：
      | 準備中 |
      | 進行中 |
      | 暫停 |
      | 完成 |
      | 取消 |
    
    當我選擇將狀態更新為"暫停"
    而且我填寫原因"等待客戶提供額外資料"
    而且我點擊"確認更新"按鈕
    那麼系統應該請求確認
    當我確認狀態更新
    那麼系統應該通知合作方"合作狀態已更新為『暫停』"
    而且更新合作狀態
    
    當我嘗試將狀態更新為"取消"
    那麼系統應該顯示警告訊息，說明此操作需要雙方確認
    當我提交取消請求並說明原因
    那麼系統應該通知對方"『合作方名稱』請求將合作狀態變更為『取消』"
    
    當對方拒絕狀態變更請求
    那麼系統應該通知我"狀態變更請求未獲通過"
    而且合作狀態應保持不變

  @issue-management
  場景: 問題與風險管理
    假設我正在查看合作"活動策劃合作"的詳情頁面
    
    當我點擊"記錄問題"按鈕
    那麼系統應該顯示問題記錄表單
    
    當我填寫以下問題信息：
      | 問題標題 | 場地容納人數不足 |
      | 嚴重程度 | 高 |
      | 問題描述 | 原定場地最多容納50人，但預計將有80人參加 |
      | 預期解決日期 | 2025-06-10 |
      | 負責方 | 社團方 |
    而且我點擊"提交問題"按鈕
    那麼系統應該顯示"新問題已登記：『場地容納人數不足』"確認訊息
    而且通知負責方"新問題已登記：『場地容納人數不足』"
    
    當我查看問題列表
    那麼我應該看到所有已記錄的問題及其狀態
    
    當我將問題標記為已解決
    而且填寫解決方案"已更換為可容納100人的大型場地"
    那麼系統應該更新問題狀態為"已解決"
    而且通知雙方"『場地容納人數不足』已標記為已解決"
    
    當問題超過預期解決日期仍未解決
    那麼系統應該顯示提醒"『場地容納人數不足』已超過預定解決日期"
    而且向負責方發送提醒通知
