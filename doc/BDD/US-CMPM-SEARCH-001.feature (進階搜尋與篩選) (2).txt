# language: zh-TW

@US-CMPM-SEARCH-001 @search-feature @P1
功能: 進階搜尋與篩選
  身為平台使用者
  我想要能夠使用多種條件和關鍵字進行精確的需求搜尋和篩選
  以便在眾多資訊中快速找到最符合我需求或可提供協助的內容

  背景:
    假設我已成功登入社團企業媒合平台
    而且系統中已存在多筆不同類型的需求和資源

  @keyword-search
  場景大綱: 關鍵字搜尋
    當我在全站搜尋框中輸入"<關鍵詞>"
    而且我點擊搜尋按鈕
    那麼系統應該<結果>

    例子:
      | 關鍵詞                   | 結果                                                    |
      | 設計                     | 顯示標題、內容或標籤中包含"設計"的所有項目，並高亮關鍵詞 |
      | 程式 開發                | 顯示同時包含"程式"和"開發"的所有項目                    |
      | "資訊安全"               | 顯示精確包含短語"資訊安全"的所有項目                    |
      | 設計+宣傳                | 顯示同時包含"設計"和"宣傳"的所有項目                    |
      | 行銷 -社群               | 顯示包含"行銷"但不包含"社群"的所有項目                  |
      | a                        | 顯示"請輸入至少2個字元"提示訊息                         |
      |                          | 顯示"請輸入搜尋關鍵詞"提示訊息                          |

  @search-suggestions
  場景: 搜尋建議功能
    當我開始在搜尋框中輸入"設計"
    那麼系統應該在輸入過程中顯示搜尋建議下拉列表
    而且搜尋建議應該包含：
      | 熱門搜尋詞 |
      | 我的歷史搜尋 |
      | 相關標籤 |
      | 自動完成建議 |
    
    當我在建議列表中點擊"設計思考工作坊"
    那麼系統應該立即執行該關鍵詞的搜尋
    而且應該將此關鍵詞添加到我的搜尋歷史中

  @tag-filter
  場景: 標籤與類別篩選
    當我進入搜尋結果頁面
    那麼我應該看到標籤雲和分類選項
    
    當我點擊標籤"設計"
    那麼系統應該篩選出帶有"設計"標籤的項目
    而且"設計"標籤應該顯示為已選中狀態
    
    當我再點擊標籤"平面"
    那麼系統應該篩選出同時帶有"設計"和"平面"標籤的項目
    而且系統應該顯示"已選擇2個標籤"提示
    
    當我點擊已選中的"設計"標籤
    那麼系統應該取消該標籤的篩選
    而且只顯示帶有"平面"標籤的項目
    
    當我點擊分類"活動合作"
    那麼系統應該篩選出類型為"活動合作"的項目
    
    當我選擇標籤和分類的組合篩選後，沒有符合條件的結果
    那麼系統應該顯示"沒有符合所選標籤的內容"訊息
    
    當我點擊"清除所有篩選"按鈕
    那麼系統應該移除所有已選擇的篩選條件
    而且顯示"已清除所有篩選條件"確認訊息

  @multi-condition-filter
  場景: 多條件組合篩選
    當我進入搜尋或瀏覽頁面
    那麼我應該看到多重篩選條件區域
    
    當我展開篩選條件面板
    那麼我應該看到以下篩選選項：
      | 發布時間 |
      | 截止時間 |
      | 需求狀態 |
      | 發布角色 |
      | 預算範圍 |
      | 地理位置 |
      | 合作類型 |
    
    當我設定以下篩選條件：
      | 發布時間 | 最近一個月 |
      | 需求狀態 | 進行中 |
      | 發布角色 | 企業 |
    那麼系統應該應用這些篩選條件
    而且顯示"已套用3個篩選條件"提示
    
    當我設定預算範圍篩選為"10000-50000"
    那麼系統應該篩選出預算在此範圍內的項目
    
    當我設定合作類型為"長期合作"
    那麼系統應該再進一步篩選出合作類型為長期的項目
    
    當我設定開始日期為"2025-06-01"和結束日期為"2025-05-01"
    那麼系統應該顯示"開始日期不能晚於結束日期"錯誤提示
    
    當我移除"需求狀態"篩選
    那麼系統應該顯示"已移除『需求狀態』篩選"確認訊息
    而且更新結果以反映新的篩選組合

  @search-result-sort
  場景: 搜尋結果排序
    當我獲取搜尋或篩選結果後
    那麼系統應該默認按"相關性"排序結果
    
    當我點擊排序選項"最新發布"
    那麼系統應該按發布時間由新到舊重新排序結果
    而且顯示"已按『最新發布』排序"確認訊息
    
    當我點擊排序選項"即將截止"
    那麼系統應該按截止日期由近到遠重新排序結果
    
    當我點擊排序選項"熱門程度"
    那麼系統應該按申請或查看次數重新排序結果
    
    當我點擊"最新發布"旁的排序方向圖標
    那麼系統應該切換排序方向為升序
    而且顯示由舊到新的結果

  @search-preference-save
  場景: 搜尋偏好保存
    當我設定了多個搜尋條件和篩選條件
    而且點擊"儲存搜尋條件"按鈕
    那麼系統應該顯示儲存對話框
    
    當我輸入搜尋條件名稱"設計相關企業需求"
    而且點擊"儲存"按鈕
    那麼系統應該顯示"搜尋條件已儲存"確認訊息
    
    當我嘗試使用已存在的名稱"設計相關企業需求"儲存另一組搜尋條件
    那麼系統應該顯示"此名稱已存在，請使用其他名稱"提示
    
    當我進入"我的搜尋條件"頁面
    那麼我應該看到我儲存的所有搜尋條件列表
    
    當我點擊已儲存的搜尋條件"設計相關企業需求"
    那麼系統應該自動套用該組條件
    而且顯示"已套用搜尋條件：設計相關企業需求"確認訊息
    
    當我編輯已儲存的搜尋條件並點擊"更新"
    那麼系統應該以新設定更新該搜尋條件
    而且顯示"搜尋條件已更新"確認訊息
